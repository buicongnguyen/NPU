Hands-on exploration

Tenstorrent tt-buda and tt-metal

My NPU learning path includes hands-on attempts with Tenstorrent's open-source stack. This is useful because it connects high-level model compilation to the lower-level kernel and memory model that determines real accelerator performance.

tt-buda

High-level model compiler path

Tried the BUDA/PyBuda path for compiling AI/ML models from frameworks such as PyTorch, TensorFlow, ONNX, and TFLite to Tenstorrent hardware. The repository is now archived, so it is best treated as historical compiler-stack experience.

tt-metal

Low-level programming model

Tried and studied the TT-Metalium route for C++ kernels, Tensix cores, explicit data movement, memory layout, tiling, circular buffers, and host-to-device execution. This is the most relevant path for understanding performance beyond a model-import API.

Bridge

Compiler to runtime thinking

The key learning point is the bridge from graph-level optimizations to hardware execution: operator decomposition, tiling, sharding, buffer placement, NOC movement, and profiling.

Next

Follow current Tenstorrent stack

Since tt-buda is archived, the practical continuation is tt-metal, TT-NN, and TT-Forge, Tenstorrent's newer MLIR-based compiler stack.

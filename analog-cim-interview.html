Choose a primary lane

Four interview tracks with a shared architecture core

Master your role deeply, then learn enough of the adjacent layers to make clean interfaces and credible tradeoffs.

Analog and mixed signal

Signal integrity becomes numerical accuracy

Prepare: MOS operation, noise, mismatch, PVT, current mirrors, integrators, comparators, ADC/DAC architectures, references, calibration, layout effects, Monte Carlo, and Verilog real-number models.

Be ready to design: a column readout with a stated range, ENOB, settling time, power budget, calibration path, and corner plan.

Architecture, RTL, and verification

Make a probabilistic primitive behave like a system block

Prepare: pipelines, SRAM, NoCs, dataflow, fixed-point arithmetic, power and clock domains, RISC-V/SIMD roles, performance modeling, SystemVerilog, UVM, RNM, assertions, and coverage.

Be ready to design: a tile scheduler and verification plan that covers analog latency, backpressure, saturation, faults, reset, and power transitions.

Compiler and ML systems

Turn graph semantics into fixed physical resources

Prepare: modern C++ and Python, SSA/graph IR, ONNX, MLIR/LLVM, TVM/XLA/IREE concepts, lowering, fusion, tiling, scheduling, memory allocation, quantization, error injection, and heterogeneous partitioning.

Be ready to design: an IR and cost model for array shapes, ADC phases, partial sums, local SRAM, digital fallback, and route pressure.

Silicon, product, and test

Prove the chip across units, corners, and time

Prepare: bring-up, characterization, ATE, wafer sort, yield analysis, correlation, guardbands, binning, screening, DFT/MBIST, failure analysis, reliability, automation, and statistical data analysis.

Be ready to design: a test flow that separates device defects, analog variation, digital logic faults, package effects, and model-level escapes.

Portfolio lab

Build a noisy MVM study that exposes tradeoffs

A small, honest experiment is more persuasive than a broad slide deck with ideal TOPS.

Experiment

Model the signal chain

  1. Start with an FP32 matrix-vector reference.
  2. Quantize weights and activations with explicit scales.
  3. Map signed weights to differential conductances.
  4. Add programming error, drift, read noise, IR drop, clipping, and ADC quantization one at a time.
  5. Measure MVM error and model accuracy across random seeds and time.
Mitigation study

Pay for every recovery

  1. Compare per-tensor and per-channel scaling.
  2. Add write-verify or measured-error calibration.
  3. Try noise-aware fine-tuning.
  4. Route one sensitive layer to a precise digital path.
  5. Report extra cells, cycles, conversions, digital operations, and calibration frequency.

Deliverables: a reproducible notebook or test program, one error-budget plot, one accuracy-versus-energy proxy curve, a short design memo, and a limitations section that distinguishes measured assumptions from guessed ones.

Mock technical loop

Questions and the signal a strong answer should contain

Architecture

Derive analog MVM from first principles.

State V-to-input and G-to-weight mappings, derive cell current and column sum, then list the linearity, independence, range, and readout assumptions.

Numerical design

Why is ANA8 not simply ordinary INT8?

Separate quantization error from analog programming/read error, explain calibration and analog-aware retraining, and define the digital reference.

Mapping

Map a 4096 by 4096 projection onto 1024 by 1024 arrays.

Find the 4 by 4 partition, explain broadcast and partial-sum reduction, account for signed and bit-sliced representation, then identify bottlenecks.

Converters

Choose ADC precision for a layer.

Use output distribution, signal range, noise floor, clipping budget, ENOB, area/energy, conversion phases, and model sensitivity rather than naming a bit count.

Debug

Accuracy drops after one week in the field.

Reproduce by chip and temperature, compare reference paths, separate common gain drift from per-cell drift, test recalibration, and check whether model margins changed.

Verification

Verify a digital controller around an analog array.

Define a real-number model, bounded error contract, saturation and latency behavior, randomized PVT/error distributions, assertions, coverage, and transistor-level correlation.

Compiler

Partition a transformer between analog and digital compute.

Favor dense projections for analog; evaluate outlier-sensitive early/output layers; leave softmax, normalization, dynamic indexing, and control on flexible digital units.

Evidence review

Did Nature prove accuracy is solved?

Say no as a universal claim, then cite one measured success, its baseline and mitigation, one open issue, and the end-to-end experiment needed to generalize it.

Answer frameworks

Keep technical answers structured under pressure

Explain a block

Purpose, equation, boundary, error, metric

What work does it own? What relation implements it? What enters and exits? How does it fail? Which measurement proves it works?

Design a system

Workload, mapping, budget, mitigation, validation

Fix tensor shapes and targets, place work, allocate precision/error/energy, price each mitigation, then define end-to-end tests and corners.

Debug silicon

Reproduce, localize, separate, correlate, prevent

Control variables, narrow the layer, split analog/digital and static/dynamic causes, correlate models to measurements, then add coverage or monitoring.

Questions for the team

Ask about the real engineering boundary

  • Which workload and system boundary define the current performance-per-watt target?
  • How does the compiler decide between analog arrays and digital execution for a sensitive operator or channel?
  • Which non-ideality currently consumes the largest accuracy, area, or calibration budget?
  • How are array models correlated from transistor simulation to post-silicon and model-level accuracy?
  • What calibration is factory-only, boot-time, periodic, temperature-triggered, or workload-specific?
  • How does the team measure yield and guardband model accuracy across chips rather than on a golden unit?
  • What changed most between the M1076 generation and the current hybrid analog-digital architecture?
  • What would a strong first 90 days look like for this role?

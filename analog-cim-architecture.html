Terminology

AiCM, AIMC, AiMC, and analog CIM

The research literature most often says analog in-memory computing (AIMC or AiMC). Analog compute-in-memory (analog CIM or ACIM) expresses the same central idea. This study treats AiCM as the user's name for that family. It is different from digital CIM, which performs exact bit operations inside or beside a memory array.

The physical primitive

One matrix-vector multiply in five steps

The ideal equation is compact. Every production challenge enters through the assumptions hidden underneath it.

Ideal assumption

Cells are linear and independent

Real I-V curves, source resistance, wire drop, coupling, and sneak paths make a large array input dependent.

Ideal assumption

Weights stay where programmed

Programming spread, defects, relaxation, drift, temperature, and aging change the effective conductance.

Ideal assumption

Readout preserves information

ADC range, offset, noise, resolution, and saturation place a ceiling on useful end-to-end precision.

From tensor operation to physical math

A framework MatMul becomes repeated analog matrix-vector operations

Software usually presents a matrix-matrix operation. A conductance array naturally evaluates one input vector against a resident weight matrix. Batch rows, oversized dimensions, signs, and additional bits are handled through repeated firings, parallel arrays, and digital reconstruction.

GEMM: matrix by matrix MVM: one vector by resident weights MAC: one product added to a sum Row driver: presents an activation to array rows Column readout: senses a dot-product result Partial sum: one contribution to a larger output Sequencer: hardware that times drivers, sampling, and conversion

Short answer: where does the MatMul happen?

For C[B,N] = A[B,K] x W[K,N], the compiler places blocks of W into conductance arrays. For each batch row A[b,:], input circuitry drives a vector onto array rows. Each memory cell performs the physical product ViGij; each column wire and sensing circuit accumulates those products; the ADC creates digital partial results; and nearby digital logic subtracts signed paths, combines bit planes and array partitions, applies scale and bias, and stores or routes C[b,:]. The RISC-V controller schedules this work; it does not execute the bulk analog MACs.

Design rule: the control processor should issue compact work descriptors and handle exceptional control. A dedicated sequencer should own repetitive cycle-level analog timing. If firmware must micromanage every input phase, ADC lane, and transfer, controller utilization and NoC traffic can erase the array's theoretical advantage.

Mapping a real layer

Array dimensions determine which results concatenate and which must be added

The compiler must distinguish four independent decompositions. Confusing an N shard with a K shard produces the wrong graph even when every local MVM is correct.

Split N: output columnsConcatenateEach array stores a different group of output columns and receives the same activation slice. Outputs represent distinct channels and are joined side by side.
Split K: reduction rowsAdd partial vectorsEach array stores a different K range for the same output channels. Its result is incomplete until a digital accumulator adds every K contribution.
Split B: batch or tokensRepeat or replicateReuse one resident weight mapping over time, or duplicate weights onto additional arrays to process independent vectors concurrently.
Split precisionShift, scale, and addSpatial weight slices or temporal input phases produce weighted fragments. Digital reconstruction aligns each fragment before summation.

2Slice and broadcast one input rowSplit A[b,:] into K0, K1, and K2. Broadcast each slice to the four arrays in its K row.

6Handle B=2Repeat for the second row or use a replicated weight mapping. Then stack both result rows into C[2,4096].

Convolution

Filters become matrix columns

Flatten each receptive-field patch across kernel height, kernel width, and input channels into a K-vector. Map output-channel filters to N columns. Stream patches as repeated MVM inputs, then reshape output vectors back to spatial feature maps.

Transformer

Linear projections dominate the analog-friendly part

Q, K, V, output, and MLP projections are dense MatMuls and can use the same decomposition. Attention score MatMul, softmax, normalization, dynamic KV-cache access, and control generally require the digital companion or another engine.

Mythic's mapping patent describes partitioning coefficients across multiple arrays, applying inputs, combining outputs, positive/negative paths, multi-row inputs, and serial multi-step input handling. NeuRRAM independently demonstrates the same system need: layers larger than an array are segmented across cores, with intermediate buffers and partial-sum accumulation outside the array in the reported prototype. Mythic case study

The disclosed M1076 tile separates bulk analog math from digital orchestration

The product brief says every tile is driven by a Mythic ACE containing flash weight storage and analog matrix-multiplication circuits, beside a RISC-V nano-processor, SIMD processor, SRAM, and NoC router. The exact internal command and reconstruction boundary is not public, so the responsibilities below distinguish documented blocks from the architectural role they are best suited to perform.

Mythic ACE Embedded flash stores resident weights Cells and columns perform bulk MVM Input drive, sensing, and ADC form the mixed-signal boundary
Local SRAMPer-inference activations, outputs, descriptors, and temporary state
16-bit SIMDDigital vector and post-processing work; exact operator allocation is not public
32-bit RISC-VTile sequencing and control, not the bulk array MACs
NoC routerMoves activations, results, commands, and events between tiles

Known versus current roadmap

The M1076 block structure and first-generation product figures are publicly documented. Mythic's 2025-2026 claims for 120 TOPS/W, LLM scale, and roughly 100x system advantages are company roadmap and internal benchmark statements. They should not be presented as independently reproduced measurements.

Mythic's May 2026 Videantis acquisition also makes the hybrid boundary explicit: analog for dense AI matrix operations, unified digital compute for attention, signal processing, codecs, SLAM, control, and programmability.

Hardware-software co-design

The model reaches the array through a compiler pipeline

1TrainPyTorch or another standard framework
2QuantizeFP32 to ANA8; choose scales and ranges
3HardenQAT or analog-aware retraining where needed
4PartitionMap matrices and digital operators to tiles
5ScheduleAllocate buffers, routes, and dataflow tasks
6ValidateMeasure accuracy, latency, power, and corners

The correct mental model is an analog MVM accelerator inside a digital system. The compiler and calibration stack determine tensor ranges and placement; the analog array supplies dense parallel arithmetic; digital units preserve generality and precision where the array is a poor fit.
